### E2E Flow Verification

**Primary flow: Watchlist → Scan → Score → Alert** — COMPLETE (via CLI `run`)

The full pipeline is wired end-to-end:
1. User adds card via CLI `watch` or server `POST /api/watch` → resolver → watchlist.add() with grade → JsonStorageAdapter persists
2. Scheduler `runOnce()` calls watchlist.listActive() → prioritizes oldest
3. Scanner.scan() → eBay Browse API
4. PriceEngine.getFMV() → null guard prevents crash
5. DealScorer.scoreMany() → thin-market downgrade active
6. Filter by minDealScore and targetPrice
7. TelegramAlerts.sendDealAlert() → grammy Bot → Telegram
8. Dedup persisted at both alert and scheduler layers
