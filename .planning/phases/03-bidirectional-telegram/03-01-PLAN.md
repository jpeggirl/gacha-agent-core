    4. Add `startPolling()` and `stopPolling()` methods to the TelegramAlerts class (after `clearDedupeCache`):
       ```typescript
       startPolling(): void {
         this.bot.on('message', async (ctx) => {
           await ctx.reply('Got it — more commands coming soon.');
         });

         this.bot.catch((err) => {
           const e = (err as { error: unknown }).error;
           console.error('[Telegram] Handler error:', e);
           // Swallow — do not re-throw; keeps polling alive on handler errors.
           // Polling loop errors (network failure) crash the process via grammy default — intentional.
         });

         void this.bot.start({
           allowed_updates: ['message'],
           drop_pending_updates: true,
           onStart: (info) =>
             console.log(`[Telegram] Polling started as @${info.username}`),
         });
       }

       async stopPolling(): Promise<void> {
         await this.bot.stop();
       }
       ```

       Key constraints:
       - `bot.on('message')` MUST be registered before `bot.start()`
       - `bot.catch()` MUST be registered before `bot.start()`
       - `void this.bot.start(...)` — do NOT await; the Promise never resolves while running
       - `drop_pending_updates: true` — discards stale messages from before boot (avoids noisy acknowledgments on restart)
       - `startPolling()` is synchronous and returns void; the polling runs in the background
       - `stopPolling()` is async (bot.stop() is async)

    Note on GrammyError/HttpError: The `bot.catch` handler receives a grammy-wrapped error object. Use a type cast `(err as { error: unknown }).error` rather than importing GrammyError/HttpError — those imports are not needed for simple error logging.
