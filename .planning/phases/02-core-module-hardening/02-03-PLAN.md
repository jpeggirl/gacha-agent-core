Purpose: The current throw-on-failure propagates up and stops the entire scheduler loop. Returning null lets the scheduler skip FMV-dependent steps and continue to the next entry. This is the critical error boundary fix for PRICE-01.

Output: Updated engine.ts with null-return error handling, engine.test.ts with MSW-mocked tests.
</objective>

<feature>
  <name>Null-return error boundary for PriceEngine</name>
  <files>src/pricing/engine.ts, src/pricing/engine.test.ts</files>
  <behavior>
    1. getFMV(card, 10) with API returning valid pricing data
       -> FairMarketValue { fmv: median, prices: [...], populationCount: N }

    2. getFMV(card, 10) called twice within 30 minutes
       -> Second call returns cached value (no network request)

    3. getFMV(card, 10) when API returns 503
       -> null (not thrown error)
       AND console.error is called with "[PriceEngine] API error for {card.name} PSA {grade}: ..."

    4. getFMV(card, 10) when API returns { success: false, error: "Not found" }
       -> null
       AND console.error with "[PriceEngine] No data for {card.name} PSA {grade}: Not found"

    5. getMultiGradeFMV(card, [9, 10]) when grade 10 fails
       -> Map with only grade 9 result (grade 10 silently skipped)
  </behavior>
  <implementation>
    1. Change return type of `getFMV()` from `Promise<FairMarketValue>` to `Promise<FairMarketValue | null>`.

    2. Replace the throw in the "no data" case with:
       ```typescript
       if (!response.success || !response.data) {
         console.error(`[PriceEngine] No data for ${card.name} PSA ${grade}: ${response.error ?? 'Unknown'}`);
         return null;
       }
       ```

    3. Wrap the entire body (after cache check) in try/catch:
       ```typescript
       try {
         const response = await this.fetchPsaPricing(card.id, grade);
         // ... existing logic ...
       } catch (err) {
         const msg = err instanceof Error ? err.message : String(err);
         console.error(`[PriceEngine] API error for ${card.name} PSA ${grade}: ${msg}`);
         return null;
       }
       ```

    4. Tests use MSW to mock the PokemonPriceTracker PSA pricing endpoint. Each test case sets up appropriate handlers for success/failure/empty scenarios.

    5. For cache test: Make two calls in sequence, verify MSW handler was only called once (using `server.events` or checking response).
  </implementation>
</feature>
