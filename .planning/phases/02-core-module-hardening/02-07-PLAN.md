  <behavior>
    1. runOnce() with no active watchlist entries -> logs "No active entries", no scan

    2. runOnce() with 2 entries, one never scanned, one scanned 1 hour ago
       -> Never-scanned entry processed first (prioritization)

    3. runOnce() when PriceEngine.getFMV() returns null for an entry
       -> Entry is marked as scanned, no scoring attempted, scheduler continues to next entry

    4. runOnce() when sentAlertKeys already contains an alert key
       -> That alert is not re-sent

    5. Scheduler restart: Create scheduler A, add alert key, stop A, create scheduler B with same storage
       -> scheduler B's sentAlertKeys contains the key from A

    6. Grade resolution: entry with grade=10 uses 10; entry with grade=undefined and metadata.grade=8 uses 8; entry with neither uses 9

    7. Rate limits: after ebayDailyLimit reached, scheduler stops scanning (logs limit reached)
  </behavior>

    2. **Lazy-load sentAlertKeys from storage:**
    ```typescript
    private static readonly SENT_ALERT_KEYS_STORAGE_KEY = 'scheduler:sent-alert-keys';
    private storage: StorageAdapter;
    private sentAlertKeysLoaded = false;

    private async ensureSentAlertKeysLoaded(): Promise<void> {
      if (this.sentAlertKeysLoaded) return;
      const stored = await this.storage.get<string[]>(ScanScheduler.SENT_ALERT_KEYS_STORAGE_KEY) ?? [];
      this.sentAlertKeys = new Set(stored);
      this.sentAlertKeysLoaded = true;
    }

    private async markAlertSent(key: string): Promise<void> {
      this.sentAlertKeys.add(key);
      await this.storage.set(ScanScheduler.SENT_ALERT_KEYS_STORAGE_KEY, Array.from(this.sentAlertKeys));
    }
    ```
