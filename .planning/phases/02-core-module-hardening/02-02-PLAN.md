<feature>
  <name>Two-tier confidence gate for card resolution</name>
  <files>src/card-resolver/resolver.ts, src/card-resolver/resolver.test.ts</files>
  <behavior>
    Expected behavior in testable terms:

    1. resolve("Charizard Base Set 1st Edition") with API returning matchScore 0.92
       -> { success: true, bestMatch: {...}, needsDisambiguation: false }

    2. resolve("Charizard") with API returning matchScore 0.78
       -> { success: false, bestMatch: undefined, needsDisambiguation: true, disambiguationReason: "Confidence 78% — below auto-proceed threshold", candidates: [...] }

    3. resolve("asdfghjkl") with API returning matchScore 0.40
       -> { success: false, bestMatch: undefined, needsDisambiguation: false, candidates: [...] }

    4. resolve("Charizard") when API returns 503
       -> { success: false, candidates: [], needsDisambiguation: false }
       AND console.error is called (error is surfaced, not silently swallowed)

    5. resolve("Charizard") when API returns empty matches
       -> { success: false, candidates: [], needsDisambiguation: false }

    Thresholds:
    - AUTO_PROCEED_THRESHOLD = 0.85
    - DISAMBIGUATE_THRESHOLD = 0.70
  </behavior>
  <implementation>
    1. Replace `CONFIDENCE_THRESHOLD = 0.7` with two constants:
       ```typescript
       const AUTO_PROCEED_THRESHOLD = 0.85;
       const DISAMBIGUATE_THRESHOLD = 0.70;
       ```

    2. Update resolve() logic:
       ```typescript
       const bestCandidate = candidates[0];
       const confidence = bestCandidate?.confidence ?? 0;
       const needsDisambiguation = confidence >= DISAMBIGUATE_THRESHOLD && confidence < AUTO_PROCEED_THRESHOLD;
       const autoProceeds = confidence >= AUTO_PROCEED_THRESHOLD;

       return {
         success: autoProceeds,
         bestMatch: autoProceeds ? bestCandidate?.card : undefined,
         candidates,
         originalQuery: query,
         needsDisambiguation,
         disambiguationReason: needsDisambiguation
           ? `Confidence ${Math.round(confidence * 100)}% — below auto-proceed threshold`
           : undefined,
       };
       ```

    3. Improve error handling in catch block — log the error before returning:
       ```typescript
       catch (error) {
         const msg = error instanceof Error ? error.message : String(error);
         console.error(`[CardResolver] Failed to resolve "${query}": ${msg}`);
         return { success: false, candidates: [], originalQuery: query, needsDisambiguation: false };
       }
       ```

    4. Tests use MSW to mock the PokemonPriceTracker API at network level. Each test sets up a handler returning the appropriate matchScore to test each confidence tier.
  </implementation>
</feature>
