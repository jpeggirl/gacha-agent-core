  <files>src/server.ts</files>
  <action>
    The server's `main()` function currently creates: `storage`, `resolver`, `watchlist`, `priceEngine`, `dealScorer`, and conditionally `scanner`. It needs to also create `TelegramAlerts` and `ScanScheduler` when credentials are present, and start/stop the scheduler with the server process.

    Step 1 — Add imports at the top of server.ts (after existing imports):
    ```typescript
    import { TelegramAlerts } from './alerts/telegram.js';
    import { ScanScheduler } from './scheduler/scan-scheduler.js';
    ```

    Step 2 — In `main()`, after the `scanner` conditional block and before `createServer(...)`, add:
    ```typescript
    let scheduler: ScanScheduler | null = null;
    if (config.ebay && config.telegram) {
      const alerts = new TelegramAlerts(config, storage);
      scheduler = new ScanScheduler(
        config,
        watchlist,
        scanner!, // scanner is non-null when config.ebay is set
        priceEngine,
        dealScorer,
        alerts,
        storage,
      );
      scheduler.start();
      console.log('[Server] Background scheduler started');
    } else {
      console.log('[Server] Scheduler not started — eBay or Telegram credentials missing');
    }
    ```

    Note: `scanner` is guaranteed non-null when `config.ebay` is truthy because of the existing conditional. The `!` assertion is safe here.

    Step 3 — Add graceful shutdown. After `server.listen(...)` call, add:
    ```typescript
    const shutdown = () => {
      console.log('[Server] Shutting down...');
      scheduler?.stop();
      server.close(() => process.exit(0));
    };
    process.on('SIGINT', shutdown);
    process.on('SIGTERM', shutdown);
    ```

    Step 4 — Update `loadConfig()` in server.ts to include `telegram` config (currently server.ts loadConfig() omits telegram entirely). Add after the `ebay` block:
    ```typescript
    telegram: process.env.TELEGRAM_BOT_TOKEN
      ? {
          botToken: env('TELEGRAM_BOT_TOKEN'),
          defaultChatId: process.env.TELEGRAM_CHAT_ID,
        }
      : undefined,
    ```

    The `/api/scan` HTTP handler is unchanged — it still performs a one-shot scan on demand. The scheduler runs independently in the background. These are complementary paths: scheduler is for automated continuous scanning, `/api/scan` is for on-demand queries.
