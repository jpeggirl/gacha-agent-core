    Add a `remove` case to the `switch (command)` block in `src/cli.ts`, positioned after the `list` case and before the `scan` case.

    Implementation:
    ```typescript
    case 'remove': {
      const id = args[0];
      if (!id) {
        console.error('Usage: gacha remove <entry-id>');
        process.exit(1);
      }
      const removed = await watchlist.remove(id);
      if (!removed) {
        console.error(`No watchlist entry found with ID: ${id}`);
        process.exit(1);
      }
      console.log(`Removed watchlist entry: ${id}`);
      break;
    }
    ```

    Design decisions (per user decision: no confirmation prompt, Claude's discretion on identifier/not-found):
    - Identifier: entry ID (UUID shown by `gacha list`). Consistent with server DELETE /api/watchlist/:id.
    - Not-found behavior: error message + exit(1). Silent no-op would hide typos on UUID strings.
    - No alias — `remove` is the canonical name (matches server-side vocabulary). No `unwatch` alias needed.

    Also update `printHelp()` to add the remove command:
    - Add line: `  remove <id>                     Remove a watchlist entry by ID`
